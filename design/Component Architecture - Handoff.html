Omaha Edge — mobile PLO advisor

A React (UMD + in-browser Babel) prototype. Every screen is a controlled Body component with no chrome of its own; a shell composes them inside one device frame. This document lists every module, component, prop, callback, and the shared state contract.

Stack & conventions

RuntimeReact 18.3.1 UMD + ReactDOM, Babel standalone (type="text/babel"). No build step.
Module sharingEach Babel script has its own scope. Cross-file sharing is via window.* (prefixed PK* / pk*). Import order matters.
StylingPer-module injected <style> with a guard id (pk-kit, pk-coach, pk-study, pk-journal, pk-shell, plus pk-app in play). Tokens are CSS vars on :root.
Device frame390×844 .pk-screen inside a 414×868 .pk-bezel, JS-scaled to viewport. Owned by the shell/wrappers — never by a Body.

The Body / Wrapper split

Body = header + content only, fully controlled by props. Wrapper (*App) = owns the bezel, status bar, tab bar, local state + overlays, and renders one Body. The unified shell renders all three Bodies and keeps them mounted for instant tab switches.

PlayBody → PlayApp StudyBody → StudyApp JournalBody → JournalApp All 3 → UnifiedApp

01 Module map & load order

Load top-to-bottom. Lower modules depend on window.* exports of the ones above. The standalone *.html files load poker-kit → coach → <that screen>; the unified prototype loads everything.

poker-kit.jsxdeps: React
Foundation — design tokens + CSS, card/atom components, the PLO advisor, and the probability-matrix data model. Pure; depends on nothing else.
coach.jsxdeps: poker-kit
Shared AI Coach sheet + BYOK first-run. Context-aware (hand / play / matrix).
play-app.jsxdeps: React · coach (overlay)
Play mode. Self-contained atoms + advisor copy. Exports PlayBody, PlayApp, pkBuildHandEntry.
study-app.jsxdeps: poker-kit · coach
Study mode (Scenarios + Matrix). Exports StudyBody, StudyApp.
journal-app.jsxdeps: poker-kit
Hand Journal. Exports JournalBody, JournalApp.
app.jsxdeps: all of the above
Unified shell — one frame, instant tab switching, shared state, launch splash + walkthrough, cross-tab save/replay. Exports UnifiedApp.
Note for refactors

play-app.jsx predates the kit and keeps its own copies of the atoms and advise() (separate Babel scope — no collision). If consolidating, point PlayBody at the poker-kit exports and delete the local copies. Behaviour is identical today.

Full-height sheet (z-index:12) used by both Play and Study. First-run shows BYOK; thereafter the chat. Injected under pk-coach.

The context prop (drives opener + suggested replies)

// from Play (PlayBody)
{ kind: 'play', name: 'this hand', cards: [{rank,suit}×4] }   // cards omitted until 4 entered

// from Study › Scenarios

07 app.jsx unified shell

Composes the three Bodies in one frame and owns all shared state. Injected under pk-shell.

09 Cross-tab flows

  onSaveHand(pkBuildHandEntry(cards, adv))

  + toast

  'This session' group (top)

Replay → Play

  onReplay(hand)

  setCards(hand.hole)
  setMode('starter'); setTab('play')
PlayBody: shows locked read
Verified end-to-end

Launch → walkthrough → enter cards → read → swipe-save → Study/Matrix → Coach (matrix context) → Journal → expand → Replay → back in Play. All three tabs render; Coach opens from Play & Study. No console errors.
